# IDE and environment #

## IDE and editors used ##
*   Atom (project management & usual editing)
*   Vim/spacemacs/neovim (small fast/remote editing)
*   Unity (Games) ?

## Why not use IDE ##
*   Magic makes things easy but you have to learn the IDE which is equivalent
    with learning how to do the magic and automatize it yourself
*   Autocompletion is cool but you have to read the doc anyways so you better
    create snippets and have the doc easily accessible

## Atom's cons ##
*   Lacks integration of interactive terminal
*   Java autocomplete
*   Slow start time (15-20sec)

## Language support needed ##
*   C [, C++, C#]
*   Go
*   Bash, Makefile
*   Java
*   Python
*   Javasript/NodeJs
*   HTML5, CSS3, bootstrap
*   Markdown for notes

## IDE Functionalities ##
*    Customizable (keybinding, plugins)
*    Ergonomic navigation with the use of normal shortcuts :
     C-c, C-v, C-x, C-z, c-y arrows etc.)
*    Projects management
*    Command line/Script activation (build, run, config) shortcuts
*    Remote synchronization via sftp/scp
*    Easy access to API doc/Autocomplete + snippets + Comment generation
*    Linter debug on the fly

## Editor ##
*   UTF-8 standard encoding + conversion
*   Linux EOL conversion
*   Line numbering
*   Trim on save
